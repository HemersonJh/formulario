Perguntas
  1. Na tag <input> com atributo type="text", para que serve o atributo "placeholder"?

    O placeholder atributo especifica uma breve dica que descreve o valor esperado de um campo de entrada (por exemplo, um valor de amostra ou uma breve descrição do formato esperado). A dica curta é exibida no campo de entrada antes que o usuário insira um valor. Nota: O placeholder atributo funciona com os seguintes tipos de entrada: texto, pesquisa, url, tel, email e senha.


  2. Na tag <input>: com atributo type="radio" ou type="checkbox", para que serve o atributo "checked"?

    O checked atributo é um atributo booleano. Quando presente, especifica que um <input> elemento deve ser pré-selecionado (marcado) quando a página for carregada. O checked atributo pode ser usado com <input type="checkbox"> e <input type="radio">. O checked atributo também pode ser definido após o carregamento da página, com JavaScript.

  3. Na tag <input> para que serve o atributo "value"?

    O atributo input Value especifica um valor inicial para um campo de entrada

  4. Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?

    O grupo de rádios deve compartilhar o mesmo nome (o valor do nameatributo) para ser tratado como um grupo. Depois que o grupo de opções for criado, selecionar qualquer botão de opção nesse grupo desmarcará automaticamente qualquer outro botão de opção selecionado no mesmo grupo. Você pode ter quantos grupos de rádio quiser em uma página, desde que cada grupo tenha seu próprio nome

  5. Para que serve a tag <label>? Para que serve o atributo "for" nessa tag?

    Tag Label: é importante para os campos de formulários. Ela especifica qual o "rótulo" do input (a que se refere o input, como por exemplo envolvê-la em um texto “Nome completo”), e ajuda na experiência do usuário durante a utilização e preenchimento do formulário. agrupar e organizar determinados campos de um formulário, que são as tags <legend> (pode ser traduzido como uma legenda, no sentido de descrição) e a <fieldset> (conjunto de campos). A sintaxe da tag fieldset é: <fieldset> Conjunto de campos que queremos agrupar aqui <input...> </fieldset> Atributo “For”: O atributo for faz um tipo de link entre a label e o input . Logo o for tem que apontar para o ID do input . Outra forma de uso é que vc pode fazer vários tipos de customização pelo CSS escondendo o input default do browser por exemplo e usando a própria label como se fosse o botão.

  6. Na tag <option> para que serve o atributo "selected"?

    O elemento option permite o atributo selected , que, quando marcado, deixa a opção correspondente pré-selecionada.

  7. Nas tags de formulário para que serve o atributo "required"?

    O required atributo é um atributo booleano.
    Quando presente, especifica que um campo de entrada deve ser preenchido antes do envio do formulário.

    Funciona com os seguintes tipos de entrada:
    • Texto
    • Pesquisa
    • Url
    • Telefone
    • E-mail
    • Senha
    • Seletores de data
    • Número
    • Caixa de Seleção
    • Rádio
    • Arquivo

  8. Para que serve a tag <textarea>?

    Define um controle de entrada de texto multilinha.É usado em um formulário para coletar entradas do usuário, como comentários ou avaliações.

    Uma área de texto pode conter um número ilimitado de caracteres e o texto é renderizado em uma fonte de largura fixa (geralmente Courier). O tamanho de uma área de texto é especificado pelos atributos cols e rows(ou com CSS).
    O name atributo é necessário para referenciar os dados do formulário após o envio do formulário (se você omitir o nameatributo, nenhum dado da área de texto será enviado). O id atributo é necessário para associar a área de texto a um rótulo. Dica: Sempre adicione a tag <label> para melhores práticas de acessibilidade!          

  9. Na tag <form> para que serve o atributo "action"?

    O manipulador de formulário é especificado no ACTION atributo do formulário.O manipulador de formulário normalmente é um arquivo no servidor com um script para processar dados de entrada.

  10. Na tag <form> para que serve o atributo "method"?

    O methodatributo especifica o método HTTP a ser usado ao enviar os dados do formulário. Os dados do formulário podem ser enviados como variáveis de URL (com method="get") ou como pós-transação HTTP (com method="post").O método HTTP padrão ao enviar dados de formulário é GET.

  11. Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag <form>?

    O methodatributo especifica o método HTTP a ser usado ao enviar os dados do formulário. Os dados do formulário podem ser enviados como variáveis de URL (com method="get") ou como pós-transação HTTP (com method="post"). O método HTTP padrão ao enviar dados de formulário é GET. Notas sobre GET:

  12. Acrescenta os dados do formulário ao URL, em pares nome/valor
    1. NUNCA use GET para enviar dados confidenciais! (os dados do formulário enviado ficam visíveis no URL!)
    2. O comprimento de um URL é limitado (2.048 caracteres)
    3. Útil para envios de formulários onde um usuário deseja marcar o resultado
    4. GET é bom para dados não seguros, como strings de consulta no Google Notas sobre POST:
    5. Acrescenta os dados do formulário ao corpo da solicitação HTTP (os dados do formulário enviado não são mostrados na URL)
    6. POST não tem limitações de tamanho e pode ser usado para enviar grandes quantidades de dados.
    7. Os envios de formulários com POST não podem ser marcados como favoritos Dica: Sempre use POST se os dados do formulário contiverem informações confidenciais ou pessoais!

  13. Qual valor deve ser informado no atributo "method" na tag <form> em um formulário que será usado para fazer upload de arquivos?

    Para um formulário que será usado para fazer upload de arquivos, o valor a ser informado no atributo "method" na tag <form> deve ser "POST". O método "POST" é comumente usado para enviar dados do formulário, incluindo arquivos, para o servidor. Com esse método, os dados são enviados no corpo da solicitação HTTP, em vez de serem anexados à URL como no método "GET". Portanto, o atributo "method" ficaria assim:
    html <form method="POST" action="/upload" enctype="multipart/form-data"> <!-- Campos do formulário --> <input type="file" name="arquivo"> <input type="submit" value="Enviar"> </form>
     Observe que também é necessário adicionar o atributo "enctype" com o valor "multipart/form-data". Isso indica ao navegador que o formulário contém dados de arquivo a serem enviados.

  14. O que as tags <fieldset> e <legend> produzem?

    <fieldset> A <fieldset> tag é usada para agrupar elementos relacionados em um formulário. A <fieldset> tag desenha uma caixa ao redor dos elementos relacionados. Dica: A tag <legend> é usada para definir uma legenda para o <fieldset>elemento. <legend> A <legend> tag define uma legenda para o elemento <fieldset> . Atributos Globais A <legend> tag também suporta atributos globais em HTML . Atributos do Evento A <legend> tag também suporta atributos de evento em HTML .
